Calorie Tracking App

A full-stack nutrition tracking application with a database-backed REST API and a modern React interface for logging meals, monitoring calories, and reviewing nutrition analytics in one focused workflow.

React REST API SQLite MySQL
Live Demo GitHub

Overview

This project is a full-stack calorie tracking application designed to help users record food intake, understand daily energy balance, and explore nutrition trends over time without the clutter of generic fitness apps.

Problem

Tracking nutrition day after day is tedious, and many popular apps overload the experience with ads, upsells, and features that get in the way of the core habit: logging what you eat and seeing the numbers clearly.

Solution

The app keeps the interface clean and task-focused while a proper database and REST API power reliable storage, queries, and analytics so calorie and macro data stay accurate and easy to revisit.

Architecture

Requests flow from the browser through the API layer into persistent storage, with analytics computed or aggregated from that data for dashboards and summaries.

Features

Core capabilities that support daily logging and longer-term insight.

Food Logging Add meals and items quickly so daily intake stays up to date with minimal friction.
Calorie Calculator Totals update as you log food so you always see progress toward your calorie target.
Nutrition Analytics Charts and summaries highlight trends in macros and energy over days and weeks.
User Profiles Personalized goals and settings persist per user through the backend and database.

My Role

End-to-end ownership of the product build, from interface to API and persistence.

  • Designed and implemented the React front end and component structure
  • Built the REST API and request/response contracts for logging and analytics
  • Modeled data and migrations for SQLite and MySQL environments
  • Wrote styling and layout for a consistent, readable UI
